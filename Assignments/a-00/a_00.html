1. Introduction

In case this tutorial is not enough we recommend w3schools.com. There you find tutorials for JavaScript and also HTML, CSS, and many other topics related to web development.

Why JavaScript?

  • Language of the web. You might need it later.
  • Easy to learn.
  • Platform independent (all you need is a text editor and a browser).

Before learning JavaScript you should know some basics about the Hyper Text Markup Language, also known as HTML. Its elements are represented by tags. Metadata is written in head, visible text in body. Here is an example of a simple HTML file: 

<!DOCTYPE html>
<html>
    <head>
        <title>Page Title</title>
    </head>
    <body>
        <h1>This is a Heading</h1>
        <p>This is a paragraph.</p>
    </body>
</html>

JavaScript code is written in between script-tags. 

<script>
    Write some JavaScript code here...
</script>

Alternatively scripts can also be written in external files and included in the html file. This is what we do here so you do not need to care about html or css. All you have to do is write some JavaScript functions.

2. Basics

Let's get started with JavaScript. First of all we will practice a bit in the console of your browser. Usually, it is opened by pressing F12 on your keyboard. If not, find help here.
You can now copy and paste all the following examples to the console and play with them.

If you are familiar with C, C++, or C# then you should not have any problems in getting used to the JavaScript syntax because it is basically the same thing. Variables are declared and defined in the following way: 

// Declaration of variables
let x, y, z;
/* Definition of variables */
x = 5;
y = 6;
z = x + y;

Statements are separated by semicolons. This is not mandatory but recommended, as a missing semicolon might cause problems. Comments are in C++ style and either start with // or are in between /**/. A major difference to C/C++ is that there is no type definition. But beware that an implicit type exists!
When declaring a new variable it has the initial value undefined. The let keyword is not mandatory but recommended for plenty of reasons!

An alternative to let is var. If you read JavaScript tutorials you might find this a lot. But since let was introduced in JavaScript I do not see any reason to use var. In general let has the behavior you want.

You can find out the type of a variable with typeof. 

let str = "I am a string";
typeof(str);

Basic types are Number (64 bit floating point number), Boolean, String, and Object. Beware of implicit casting! 

let y = 1;
typeof(y);
y = "1";
typeof(y);

You have all the common operators available and some more. 

let a = 3;
a % 2;
++a;
a++;
a += 2;
a == "7";
a === "7";

Consider that JavaScript has two different types of comparison operators. One only checks the content the other also checks the type. Usually, you want to check also the type (===).
Multiple values are stored in arrays. 

let cars = ["Saab", "Volvo", "BMW"];
let name = cars[0];
cars[0] = "Opel";
console.log(cars);          // print the whole array directly
cars.sort();
cars.length;
cars.push("Audi");          // add an element to the end
cars.splice(2,1);           // remove elements
cars[7] = "Volkswagen";     // adding at higher index numbers leaves 'holes'
let cars2 = cars            // this is only a shallow copy!!
cars2[0] = "Porsche";
console.log(cars[0]);

Functions are very powerful in JavaScript for several reasons. A function can be assigned to a variable. Furthermore, not relying on types means you can give a function any input, even other functions. 

function myFunction(p1, p2) {
    return p1 * p2; 
}
let f = myFunction;
f(3,4);
f = (p1, p2) => { return p1 * p2; };        // yes we can do lamda expressions
f = (p1, p2) => p1 * p2;                    // and we can make them really short

function evaluate(f, x1, x2, x3) {
    return [f(x1), f(x2), f(x3)];
}
evaluate(x => x * x, 1, 2, 3);

function findMax () {                       // use arguments array to access all arguments
    let max = -Infinity ;
    for (let i in arguments) {
        if ( arguments[i] > max ) max = arguments[i];
    }
    return max;
}
findMax(1, 123, 500, 115, 44);

You have also already used objects, maybe without realizing it. An array is actually an object. The point where you might have realized was when we used the implicit functions of the array. Objects are very easy to create in JavaScript. 

let car = {
    type: "Fiat",
    model: 500,
    color: "white",
    drive: function(){ /*do something*/ }
};

car.type;
car["type"];
car.drive();

// Object constructor function (outdated)
function Car (type, model, color){
    this.type = type;
    this.model = model;
    this.color = color;
    this.drive = function () { /*do something*/ };
}
let fiat500 = new Car ("Fiat", 500, "white");

An object that might be of interest for you is the Math object. It contains important functions and numbers. 

Math.PI;
Math.round(4.7);
Math.pow(8,2);
Math.sqrt(64);
Math.max();
Math.random();

I assume you know about if, switch, for, while. Some of them were used in the previous examples. If you need more information you will find plenty online.

3. Avoiding Bugs

Each function creates a new local scope. All variables declared inside a function are not accessible from outside. If you use a variable without declaring it, it will always be in global scope. That is why you should always declare variables with the let keyword!

myFunction();
// code here can use variable carName!

function myFunction() {
    carName = "Volvo";
    for(let i = 0; i < 5; ++i){
        // do something
    }
}

Talking about scope, there is a bit more to consider, namely hoisting. In JavaScript function declarations are hoisted, meaning all functions are moved to the top of the current scope.
Variable definitions are not hoisted! This is especially important for functions. 

a(5);    // this is fine
b(5);    // this will return an error
function a(x) { console.log("a = " + x) };
let b = function(x) { console.log("b = " + x) };

Some hints for debugging: The easiest way to get some information about your code might be console.log() which prints out stuff on the console. Furthermore, you can write the debugger keyword in a script which will enforce the debugger of your browser. It has the same effect as setting a breakpoint in your debugger. Talking about debugger, your browser has one. Use it!

Finally, I want to tell you some best practices which should help to avoid bugs.

  • Avoid global variables, new, and ==.
  • Beware of automatic type conversions.
  • Use a semicolon at the end of each statement.
  • Do not access arrays with named indexes.
  • Write in lower camelCase
